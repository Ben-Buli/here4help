🚀 主要端點

POST /auth/login.php

用戶登入

curl -X POST http://localhost:8888/here4help/backend/api/auth/login.php \ -H "Content-Type: application/json" \ -d '{"email": "user@example.com", "password": "password123"}'
POST /auth/refresh-token.php

刷新 Access Token

curl -X POST http://localhost:8888/here4help/backend/api/auth/refresh-token.php \ -H "Content-Type: application/json" \ -d '{"refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."}'
GET /tasks/list.php

獲取任務列表

curl -X GET "http://localhost:8888/here4help/backend/api/tasks/list.php?page=1&per_page=20" \ -H "Authorization: Bearer <token>"

🔍 請求追蹤

每個 API 請求都會生成唯一的 TraceId，用於問題追蹤和除錯：

  • 請求標頭: 可在 X-Trace-Id 標頭中傳遞自定義 TraceId
  • 回應標頭: 回應會包含 X-Trace-Id 標頭
  • 回應內容: TraceId 也會包含在回應的 JSON 中
# 請求範例 curl -X GET "http://localhost:8888/here4help/backend/api/tasks/list.php" \ -H "Authorization: Bearer <token>" \ -H "X-Trace-Id: CUSTOM123456789" # 回應標頭 X-Trace-Id: CUSTOM123456789 X-Request-Id: CUSTOM123456789

⚡ 節流限制

API 實施節流限制以防止濫用：

  • 認證端點: 5 requests / 5 minutes
  • 訊息端點: 30 requests / 1 minute
  • 檢舉端點: 3 requests / 1 hour
  • 一般端點: 100 requests / 1 minute

節流資訊會在回應標頭中提供：
